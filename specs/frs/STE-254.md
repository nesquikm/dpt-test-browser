## Requirement

Define the in-memory data model and state container for the browser's tabs. A `BrowserTab` value type holds a tab's id, URL, title, and loading flag. A `TabManagerCubit` owns the list of open tabs plus a pointer to the active tab and exposes the operations the rest of the app uses to mutate that state. No webview, no UI — pure Dart logic, fully unit-testable.

This FR unblocks the webview-adapter and shell-UI FRs.

## Acceptance Criteria

- AC-STE-254.1: `BrowserTab` is a Freezed data class with required `String id` + `Uri url`, optional `String? title`, and `bool isLoading` defaulting to `false`. Two tabs with identical fields are `==`-equal.
- AC-STE-254.2: `TabManagerCubit` initial state has `tabs == []` and `activeTabId == null`.
- AC-STE-254.3: `openTab(Uri url)` appends a new tab with a unique id, sets it as the active tab, and emits the resulting state.
- AC-STE-254.4: `closeTab(String id)` removes the tab with that id. If the closed tab was active, the previously-active-or-adjacent tab becomes active; if the list empties, `activeTabId` becomes `null`. No-op if `id` is not present.
- AC-STE-254.5: `selectTab(String id)` sets `activeTabId` to `id` and emits. No-op if `id` is not present.
- AC-STE-254.6: `navigate(String id, Uri url)` updates the matching tab's `url`. No-op if `id` is not present.
- AC-STE-254.7: `setLoading(String id, bool loading)` updates the matching tab's `isLoading` flag. No-op if `id` is not present.
- AC-STE-254.8: gate (`fvm flutter analyze && fvm flutter test`) is clean.

## Technical Design

- `lib/tabs/tab.dart` — `BrowserTab` Freezed class.
- `lib/tabs/tab_manager_state.dart` — `TabManagerState` Freezed class with `List<BrowserTab> tabs` and `String? activeTabId`.
- `lib/tabs/tab_manager_cubit.dart` — `TabManagerCubit extends Cubit<TabManagerState>`; methods listed in the ACs.
- New deps: `flutter_bloc`, `freezed`, `freezed_annotation`, `build_runner` (dev), `uuid`. Codegen via `dart run build_runner build --delete-conflicting-outputs`.
- No persistence in this FR — state lives only in memory (persistence is a later FR).

## Testing

- `test/tabs/tab_test.dart` — Freezed equality + copyWith roundtrip.
- `test/tabs/tab_manager_cubit_test.dart` — `bloc_test` per cubit method covering happy + no-op-on-missing-id paths.
- Tab id uniqueness is structural (uuid) — covered by an open-two-tabs test asserting `tabs[0].id != tabs[1].id`.

## Notes

- Active-tab successor on close: previous-by-index, falling back to first remaining tab. Matches typical browser UX; documented in cubit dartdoc.
- `Uri` (not `String`) for URLs — parsing happens at the address-bar boundary in a later FR.
